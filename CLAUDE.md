# FPT Cloud Documentation Project

## Structure

Docusaurus site using the **standard Docusaurus i18n layout** with 3 locales
(en / vi / ja). English is the source of truth; vi and ja are translations
that mirror the en folder tree file-for-file.

```
docs/                                            → English (default locale)
i18n/vi/docusaurus-plugin-content-docs/current/  → Tiếng Việt
i18n/ja/docusaurus-plugin-content-docs/current/  → 日本語
```

URL scheme (with `trailingSlash: true`):

- English: `/docs/<path>/`
- Vietnamese: `/vi/docs/<path>/`
- Japanese: `/ja/docs/<path>/`

All 3 trees share the **same folder hierarchy and the same filenames**,
derived from the original manual sidebar menu. Translations are matched by
relative path + filename — NEVER by localized filenames or frontmatter `id`.

Top-level folders are category wrappers:

- `billing/`
- `fpt-ai-factory/`, `fpt-appsec/`, `fpt-cloud-backup-and-dr/`,
  `fpt-cloud-server/`, `fpt-container/`, `fpt-data-platform/`,
  `fpt-database-engine/`, `fpt-devsecops-services/`, `fpt-integration/`,
  `fpt-monitoring/`, `fpt-network/`, `fpt-security/`, `fpt-storage/`
- `gpu-container/`, `gpu-virtual-machine-en/`, `managed-gpu-cluster-kubernetes/`,
  `managed-gpu-cluster-kubernetes-ja/`, `metal-cloud-ja/`
- `support/`
- `intro.md` (site root at `/docs/intro/`)

### `_category_.json`

Every category folder has a `_category_.json` with localized label, position,
and a unique `key` (required for autogenerated sidebars under i18n — Docusaurus
uses `key` as the translation key to disambiguate categories sharing the same
label):

```json
{
  "label": "Quick Starts",
  "position": 2,
  "collapsible": true,
  "collapsed": true,
  "link": { "type": "generated-index" },
  "key": "fpt-cloud-server-tagging-quick-starts"
}
```

Only `docs/**/_category_.json` needs the `key` field. i18n translations apply
on top of the default-locale structure and inherit the same keys.

### Sidebar

A single `sidebars.js` with one autogenerated entry drives the menu for all
3 locales:

```js
module.exports = {
  tutorialSidebar: [{ type: "autogenerated", dirName: "." }]
};
```

The menu structure comes entirely from the `docs/` folder tree +
`_category_.json` files. Never hand-edit `sidebars.js`. Translations of
category labels are picked up automatically from the `i18n/` tree when the
corresponding `_category_.json` file exists there.

## Adding a New Module

1. Decide the breadcrumb in the menu
   (e.g. `FPT Cloud Server > New Module > Quick Starts > ...`)
2. Create folders in all 3 locales with **the same paths and filenames**:
   - `docs/fpt-cloud-server/new-module/`
   - `i18n/vi/docusaurus-plugin-content-docs/current/fpt-cloud-server/new-module/`
   - `i18n/ja/docusaurus-plugin-content-docs/current/fpt-cloud-server/new-module/`
3. Add `_category_.json` for the module folder in each locale (localized
   `label` per locale; `key` only in `docs/`)
4. Place `.md` files inside (optionally in sub-folders like `quick-starts/`,
   `tutorials/`). **Filenames must match across all 3 locales.**
5. Add `_category_.json` for each sub-folder with localized label + position
6. `npm run build` — autogenerate picks it up

No sidebar file edits needed.

## Frontmatter Conventions

Minimum for each `.md`:

```yaml
---
title: "Page Title"           # required (or H1 in body)
sidebar_label: "Short Label"  # optional; defaults to title
sidebar_position: 3           # optional; controls order within folder
---
```

- **Do not** set `id:` explicitly — let Docusaurus infer it from the filename.
  Explicit `id` fields were the source of translation-key collisions during
  the i18n standard migration.
- **Do not** use `slug: "/..."` overrides — URLs should be derived from folder
  structure.

## Docsmith Integration

External tool: `docsmith` plugin (`/docsmith start <product>`) generates pure
markdown files. To integrate its output:

1. Run `/docsmith draft <product>` in a workspace
2. Copy resulting `.md` files into the appropriate folder in this project
   (e.g. `docs/fpt-cloud-server/new-module/quick-starts/` for English)
3. For translations, drop the vi/ja versions into the parallel i18n paths
   using the **same filenames** as the English originals
4. Add a `_category_.json` if creating a new sub-category
5. `npm run build` — done

Docsmith output is pure markdown with H1 headings, no frontmatter. Docusaurus
infers title from the H1 automatically. No script needed.

## Build

```bash
npm run start       # dev server (en) on localhost:3000
npm run start -- --locale vi   # dev server for vi
npm run start -- --locale ja   # dev server for ja
npm run build       # production build (all 3 locales)
npm run serve       # serve the built site
```

Note: `package.json` build script is pre-configured with
`NODE_OPTIONS='--max-old-space-size=8192'` because the site is large
(~5000 docs × 3 locales).

## Cross-references

Use absolute URL paths in markdown links so they resolve correctly across
locales:

- English: `/docs/fpt-cloud-server/intro/`
- Vietnamese: `/vi/docs/fpt-cloud-server/intro/`
- Japanese: `/ja/docs/fpt-cloud-server/intro/`

Prefer linking to the current-locale version (Docusaurus will prefix
automatically if you use relative markdown links like `./other-page.md`,
but absolute paths are the current convention in this repo).

## Orphan Files

Files with `draft: true` frontmatter are excluded from production build
(kept in git for historical reference). ~523 orphan files exist from the
mass migration — content not referenced by the original sidebar.

## Images

- Absolute paths only: `/img/migrated/<subfolder>/<img>.png`
- Do not use relative `./images/...` refs — they break when files move
- Images live in `static/img/migrated/` and are shared across all locales

## Migration History

- **Phase 0**: 3 separate docs plugin instances (`docs/`, `docs-vi/`,
  `docs-ja/`) with ~9000-line hand-written sidebars per locale.
- **Phase 1**: Mass restructure to match the sidebar menu hierarchy + switch
  to `{type: 'autogenerated'}` sidebars driven by `_category_.json` files.
- **Phase A (current)**: Migrated to Docusaurus i18n standard layout
  (`i18n/<locale>/docusaurus-plugin-content-docs/current/`). Deleted custom
  `LanguageSwitcher`, `langMappingPlugin`, `customDocsClient`, and swizzled
  `NavbarItem`. Single `sidebars.js`. Added `key` to all `docs/**/_category_.json`
  to avoid autogenerated-sidebar translation-key collisions.
