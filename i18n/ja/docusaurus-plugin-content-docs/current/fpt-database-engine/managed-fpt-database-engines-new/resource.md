# データベースクラスターリソースの管理

データベースクラスターのリソースには、Flavor（CPU – RAM）とデータディスク容量（Data Disk Size）が含まれます。

**データベースクラスターリソースの管理**機能により、ユーザーはデータベースクラスターに割り当てられたリソース（CPU – RAM（Flavor）、データディスク容量、ストレージポリシー）を監視・調整して、運用中のパフォーマンスと容量の要件を満たすことができます。

詳細な手順は以下の説明に記載されています。

### 1. データベースリソース情報の表示

メニューバーから**Database Platform** > **All Database**メニューまたは該当するデータベースグループのメニューを選択 > Cluster IDをクリック > **Resource**タブを選択してデータベースリソース管理画面を開きます。

[![](/img/migrated/view-resource-scaled-5cec940f.png)](/img/migrated/view-resource-scaled-5cec940f.png)

データベースのリソース情報は、Resource Informationセクションに**Current Resource**と**Available Resource**の2つの主要部分として表示されます。

- **Current Resource**：データベースの現在のリソース設定（vCPU、RAM、データディスク容量）を表示します。
- **Available Resource**：データベースの拡張に使用できる最大リソースクォータ（vCPU、RAM、データディスク容量）を表示します。

### 2. データベースリソースの調整

ユーザーはFlavorを変更してコンピュートリソース（CPU – RAM）を増減したり、利用可能なリソース制限内でデータディスク容量を増加したりできます。手順は以下の通りです。

**データベースリソース管理**画面で、**Database Resources**セクションの横にあるリソース編集アイコン（鉛筆アイコン）をクリックして**Update Database Resource**ダイアログを開きます。

[![](/img/migrated/update-resource-scaled-002f98f3.png)](/img/migrated/update-resource-scaled-002f98f3.png)

調整する情報を入力します。

- コンピュート設定（vCPU – RAM）を変更するには、ドロップダウンリストから目的の**Flavor**を選択します。
- データディスク容量を増加するには、**Data disk size (GB)**フィールドに新しい値を入力します。

**Update**をクリックして変更を適用します。

確認後、データベースクラスターのステータスは「**Resizing**」に変わり、プロセスが完了すると「**Running**」に戻ります。

リソース更新プロセスには5〜10分かかる場合があります。この間、データベースクラスターに対して他の操作を実行しないでください。

> **重要な注意事項：**
> - **リソース制限**：ユーザーはAvailable Resourceクォータ内でのみリソースを変更できます。新しい設定がこのクォータを超える場合、システムはエラーメッセージを表示し、変更を許可しません。
> - **データディスクサイズの縮小はサポートされていません**：システムはデータディスク容量の縮小をサポートしていません。ディスク容量を**増加**させることのみ可能で、現在の容量より減らすことはできません。
> - **サービスの再起動（データベースクラスターの再起動）**：コンピュートリソース設定を変更する場合、**一部のデータベースエンジンでは新しい設定を適用するためにデータベースクラスターの再起動が必要です**。再起動はシステムによって自動的に実行される場合や、リソース更新完了後にユーザーが手動で実行する必要がある場合があります。再起動プロセスにより**一時的なサービス中断が発生する場合があります**。そのため、ユーザーはその影響を最小化するために**トラフィックの少ない時間帯**にこれらの変更を行うことをお勧めします。
>
> 具体的には：
> - システムはコンピュートリソースを**縮小する際に自動的にサービスを再起動します**。
> - **コンピュートリソースを増加する**場合、システムは**自動的に再起動しません**。特に**PostgreSQLとClickHouse**については、ユーザーは新しい設定を有効にするために更新後に**追加の再起動操作を実行する**必要があります。

### 3. ストレージポリシーの調整

ストレージポリシーはデータベースのI/Oパフォーマンスと運用コストに直接影響します。ユーザーは以下の手順でこのポリシーを更新できます。

**データベースリソース管理**画面で、**Storage Policy**セクションの横にあるストレージポリシー編集アイコン（鉛筆アイコン）をクリックして**Update Storage Policy**ダイアログを開きます。

[![](/img/migrated/update-storage-policy-1-scaled-4642a33c.png)](/img/migrated/update-storage-policy-1-scaled-4642a33c.png)

ドロップダウンリストからニーズに合った新しいストレージポリシーを選択し、**Update**をクリックして変更を適用します。

ストレージポリシーの更新プロセスには約5〜10分かかる場合があります。この間、システムへの影響を避けるためにデータベースクラスターに対して他の操作を実行しないでください。

**推奨事項**：**本番**環境では、安定したパフォーマンスを確保するために**最低IOPS 4,000（4K）**のストレージポリシーを使用してください。
