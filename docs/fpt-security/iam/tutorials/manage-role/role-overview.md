# Role overview

A role defines a user's position within the organization and specifies the permissions that user can perform.

FPT IAM includes two built-in basic roles: **Super Admin** and **Readonly**. Users can create **Custom Roles** with a tailored permission set if the two basic roles do not meet their needs.

Each role has a list of permissions that determine what actions a user is allowed to perform on which resources.
