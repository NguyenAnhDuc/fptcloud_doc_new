# Manage Tag To Subnet

## **1. Assigning Tags While Creating a Subnet**
**Step 1** : In the menu, select **Subnet** , then click **Create subnet**.
[![file](/img/migrated/image-1721034781650-31f7f246.png)](/img/migrated/image-1721034781650-31f7f246.png)
**Step 2** : On the **Create subnet** page, users can select tags in the **Add tag (optional)** field to assign them to the subnet.
[![file](/img/migrated/image-1721034795338-032c129b.png)](/img/migrated/image-1721034795338-032c129b.png)
**Step 3** : Users fill in other details and click **Create subnet** to complete the process. The assigned tags will be displayed in the **Tags** field on the **Subnet** screen.
[![file](/img/migrated/image-1721034807246-4eb48b05.png)](/img/migrated/image-1721034807246-4eb48b05.png)
## **2. Managing Tags for a Subnet**
**Step 1** : In the menu, select **Subnet** , choose a subnet, then click **Actions** > **Manage tags**.
[![file](/img/migrated/image-1721034830614-0a44d416.png)](/img/migrated/image-1721034830614-0a44d416.png)
**Step 2** : A dialog box appears, allowing users to select tags to assign to the subnet.
[![file](/img/migrated/image-1721034842841-94a4bdb4.png)](/img/migrated/image-1721034842841-94a4bdb4.png)
**Step 3** : If users want to remove a tag from the subnet, they can:
  * Click the “x” next to the tag name to remove it individually.
  * Click the “x” at the end of the list box to remove all tags at once.
  * Click Save to confirm changes.

**Note: Deleting a subnet will automatically remove all associated tags, and the tag list will be updated in the Tagging screen.**
[![file](/img/migrated/image-1721034855299-a138f9c7.png)](/img/migrated/image-1721034855299-a138f9c7.png)
