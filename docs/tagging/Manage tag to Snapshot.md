# Manage Tag To Snapshot

## **1. Assigning Tags While Creating a Snapshot from an Instance**
**Step 1** : In the menu, select **Instance Management** , click **Actions** , then choose **Create snapshot**.
[![file](/img/migrated/image-1721015609867-c4ddd33b.png)](/img/migrated/image-1721015609867-c4ddd33b.png)
*_Step 2_ : In the **Create instance snapshot** dialog box, users can select tags in the **Add tag (optional)** field to assign tags to the snapshot.
[![file](/img/migrated/image-1721015621343-b9f22c97.png)](/img/migrated/image-1721015621343-b9f22c97.png)
## **2. Assigning Tags While Creating a Snapshot from a Storage Disk**
**Step 1** : In the menu, select **Storage Disk** , click **Actions** , then choose **Create volume snapshot**.
[![file](/img/migrated/image-1721015651110-79c2804c.png)](/img/migrated/image-1721015651110-79c2804c.png)
**Step 2** : In the **Create volume snapshot** dialog box, users can select tags in the **Add tag (optional)** field to assign tags to the snapshot.
[![file](/img/migrated/image-1721015663094-733cc8b6.png)](/img/migrated/image-1721015663094-733cc8b6.png)
## **3. Managing Tags for a Snapshot**
**Step 1** : In the menu, select **Snapshot**. In the **Instance snapshot** tab, click **Manage tags**.
[![file](/img/migrated/image-1721015686871-371220cc.png)](/img/migrated/image-1721015686871-371220cc.png)
**Step 2** : A dialog box will appear, allowing users to select tags to assign to the snapshot.
[![file](/img/migrated/image-1721015699377-5c976a3c.png)](/img/migrated/image-1721015699377-5c976a3c.png)
**Step 3** : Removing Tags from an Instance Snapshot
To remove a tag, click the “x” next to the tag name. Users can also click the “x” at the end of the list box to remove all tags at once. Then, click Save to confirm the changes.
**Note: When a snapshot is deleted, the system will automatically remove all associated tags and update the tag list on the Tagging screen.**
[![file](/img/migrated/image-1721015718376-1c3d39b9.png)](/img/migrated/image-1721015718376-1c3d39b9.png)
