# Change cluster configuration

The following configuration changes are available for an existing Managed FPT Kubernetes Engine cluster:

- Change cluster endpoint access
- Add a worker group
- Change internal subnet Load Balancer (CIDR)
- Edit worker group labels and taints
- Enable or disable Node Auto-repair
- Change the base worker group
- Upgrade the Kubernetes version
- Configure Cluster Autoscale
