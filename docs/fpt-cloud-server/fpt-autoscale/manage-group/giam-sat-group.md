# Monitor Group

## Step 1: Go to Autoscaling > Autoscale Group. Click the name of the group you want to monitor.

[![open group details](/img/migrated/Screenshot-2024-10-02-042416-72202578.png)](/img/migrated/Screenshot-2024-10-02-042416-72202578.png)

## Step 2: Navigate to the **Monitoring** tab.

The Monitoring page provides visual charts of the group's resource usage behavior. By monitoring regularly, you can build an effective scaling strategy for each specific service cluster.

[![monitoring dashboard](/img/migrated/screencapture-viewvm-hcm10-fptcloud-d-Ko-7d7256c2.png)](/img/migrated/screencapture-viewvm-hcm10-fptcloud-d-Ko-7d7256c2.png)

**Compute and memory**

- CPU Usage (%)
- RAM Usage (%)

**Network**

- Bandwidth (bps)
- Total Packets Transmitted (pps)

**Storage**

- Disk Read/Write
- Total Disks Throughput
