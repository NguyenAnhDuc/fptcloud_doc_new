# Access to Servers

After a GPU VM (with Ubuntu OS) is successfully created in the FPT Portal, you can access it by default using the built-in Web Console.

You can also log in externally via an SSH client or third-party software such as PuTTY or Bitvise.

## Connect to GPU VM via Web Console

The Web Console lets you control all GPU VMs on **FPT Cloud** — including those without a public IP.

In the menu, go to **Instance Management**, find the target virtual machine, and select **Console** in the **Action** section.

[![Alt text](/img/migrated/4.1-1-d8d47791.png)](/img/migrated/4.1-1-d8d47791.png)

The browser immediately opens a new window showing the server screen, giving you full control over the connected server.

[![Alt text](/img/migrated/17-a3766778.png)](/img/migrated/17-a3766778.png)

## Connect to GPU VM via SSH client

You can connect to the virtual machine via SSH using Windows built-in SSH clients such as **cmd.exe** or **PowerShell**.

**Step 1:** Open **CMD** or **PowerShell** on Windows.

**Step 2:** To connect using a password, use the following command:

  * `ssh <Username>@<IP_ADDRESS>`

[![Alt text](/img/migrated/5-2-4e4809f6.png)](/img/migrated/5-2-4e4809f6.png)

To connect using an SSH key pair, first navigate to the directory containing the SSH key file using the `cd` command:

  * `cd <PATH_TO_SSH_KEY>`

Then connect to the virtual machine using:

  * `ssh -i "<KEY_FILE>" <Username>@<IP_ADDRESS>`

**Step 3:** When connecting to the virtual machine for the first time, you are prompted to verify the host fingerprint. Type **"yes"** and press **Enter** to continue.

[![Alt text](/img/migrated/6-2-20d1ad6c.png)](/img/migrated/6-2-20d1ad6c.png)

You are now successfully connected to the server via SSH client.

[![Alt text](/img/migrated/7-2-b23b7851.png)](/img/migrated/7-2-b23b7851.png)

> 📝 **Note:** If you encounter the error **"WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!"**, use the following command to remove the previously saved host fingerprint and start over:
>
> `ssh-keygen -R "<IP_ADDRESS>"`
