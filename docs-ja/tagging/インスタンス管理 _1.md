# Instanceのタグ管理

Instanceにtagを割り当てると、仮想マシンを環境、プロジェクト、所有者別に分類でき、検索とコスト管理が容易になります。

## Instance作成時のタグ割り当て

1. メニューから **Instance Management** を選択し、**Create instance** をクリックします。

   [![Create instanceボタンのあるInstance Management画面](/img/migrated/image-1713112548052-b3294042.png)](/img/migrated/image-1713112548052-b3294042.png)

2. **Create instance** ダイアログボックスで、**Add tag (optional)** フィールドからtagを選択します。

   [![Add tagフィールドのあるCreate instanceダイアログ](/img/migrated/image-1713112563043-5f748c8d.png)](/img/migrated/image-1713112563043-5f748c8d.png)

3. その他の情報を入力し、**Create** をクリックします。

割り当てられたtagは **Instance Management** 画面の **Tags** フィールドに表示されます。

[![tagが割り当てられたInstanceリスト](/img/migrated/image-1713112579862-79f3ce78.png)](/img/migrated/image-1713112579862-79f3ce78.png)

## 既存Instanceのタグ管理

1. **Instance Management** 画面で仮想マシンを選択し、**Actions** → **Manage tags** をクリックします。

   [![Manage tagsオプションのあるActionsメニュー](/img/migrated/image-1713112615854-06a308da.png)](/img/migrated/image-1713112615854-06a308da.png)

2. ダイアログボックスが表示されます。仮想マシンに割り当てるtagを選択します。

   [![InstanceのManage tagsダイアログ](/img/migrated/image-1713112633880-63abac4e.png)](/img/migrated/image-1713112633880-63abac4e.png)

3. tagを削除するには、tag名の横にある **x** をクリックします。リストボックス末尾の **x** をクリックすると、すべてのtagを一度に削除できます。**Save** をクリックして確定します。

   [![タグ削除オプションのあるManage tagsダイアログ](/img/migrated/image-1713112743998-0feaeb65.png)](/img/migrated/image-1713112743998-0feaeb65.png)

:::warning
仮想マシンを削除すると、システムは関連するすべてのtagを自動的に削除し、Tagging画面のtagリストを更新します。
:::
