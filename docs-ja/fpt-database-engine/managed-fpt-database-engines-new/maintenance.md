# サービスメンテナンス

**定期メンテナンス**はFPT Database Engineサービスの**必須**部分であり、データベースの健全な運用状態を維持することを目的としています。これはマネージドサービスのコアコンポーネントであり、**オプション機能ではありません**。

定期メンテナンスにより、システムは以下を確保します。

- 必要なセキュリティパッチの適用。
- システムの更新と最適化。
- 長期的な安定性と運用能力の維持。

**重要な注意事項：**

メンテナンス中は、以下を含むがこれに限定されない**サーバー上のいかなる操作も行わないでください**。

- 設定の変更。
- システムの変更。
- サービスの起動または停止。

メンテナンス中にこれらの操作を行うと、パフォーマンスや安定性に影響を与えたり、予期しないエラーが発生したりする可能性があります。メンテナンスプロセスが完了した後にのみ操作を再開してください。

#### メンテナンスウィンドウ

メンテナンスウィンドウにより、ユーザーはシステムがサービス運用への影響が最も少ない時間に定期メンテナンスを実施するための優先週次時間枠を設定できます。

- 各メンテナンスウィンドウの期間：**1時間**
- ユーザーは週の曜日とメンテナンス開始時刻を指定できます。

設定しない場合、システムは**毎週土曜日23:00から始まるデフォルトのメンテナンスウィンドウを自動的に設定**します。

データベースクラスターにまだスケジュールされたメンテナンスイベントがない場合、ユーザーはメンテナンスウィンドウを変更できます。メンテナンスウィンドウはメンテナンスのタイミングを最適化しますが、サービス中断がないことを完全に保証するものではありません。サービスへの影響を最小化するため、ワークロードが少ない時間帯にメンテナンスウィンドウを設定してください。

#### メンテナンス通知

システムはユーザーがメンテナンススケジュールを積極的に把握できるよう、以下を含むメール通知を送信します。

- 予定メンテナンス時刻の**7日前通知**。
- メンテナンス開始の**前日リマインダー通知**。

通知はサービスの通知セクションで設定されたメールリストに送信されます。

#### メンテナンス中の影響と制限

メンテナンス中、サービスは以下のような一時的な影響を受ける場合があります。

- 短時間の接続中断。
- パフォーマンスの低下。
- 一部の機能の制限。

これらの影響は最小限に抑えられ、メンテナンスプロセスが完了すると解消されます。

#### メンテナンスの監視と管理

FPT Database Engineは、ユーザーがメンテナンスを積極的に管理するための以下のメカニズムを提供します。

- データベースクラスター作成時にメンテナンスウィンドウを設定。
- クラスターのデプロイ後にメンテナンスウィンドウを変更。
- 保留中のメンテナンスイベント（近日中のメンテナンスイベント）の監視。

詳細な手順については、[メンテナンスウィンドウ](/docs/ja/fpt-database-engine/managed-fpt-database-engines-new/)/セクションをご参照ください。

#### 緊急メンテナンス

重大なセキュリティパッチや重要な更新など、特定の特別な場合には、データの安全性と安定した運用を確保するため、メンテナンスウィンドウ外で緊急メンテナンスを実施することがあります。
