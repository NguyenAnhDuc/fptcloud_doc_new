# Billing Overview

## 概要
### 1. 支払い要件：
AI Factoryサービスを利用する前に、クレジットを購入するために支払いを行う必要があります。**残高はAI Factoryのすべてのサービスの支払いに使用される共有残高**であり、GPU Containerだけではないことにご注意ください。クレジットの購入が成功すると、クレジット数がTenant残高に表示されます。
### 2. 地域別ウェブサイト＆テナント：
**残高は特定のテナントと地域に紐付けられています。**
  * 現在、複数の地域またはテナントにまたがる単一残高はサポートしていません。
  * **ベトナム**のGPU H100と**日本**のGPU H200の両方をインフラとして使用したい場合は、各地域に**別々のテナント**を作成する必要があります。

_ai.fptcloud.com（ベトナム地域）またはai.fptcloud.jp（日本地域）のポータルへのサインインに成功すると、新しいテナントが自動的に作成されます。これら2つの地域のテナントは、残高とサービスリソースの点で完全に独立していることをご注意ください。_
### 3. 支払い方法：
現在、Stripe（すべての通貨に対応）とVNPay（ベトナムドンでの支払いのみ対応）によるオンライン決済をサポートしています。クレジットカード番号の保存や処理は行わず、StripeとVNPayに直接渡されます。
### 4. 料金内訳：
各コンテナにはGPU Instanceに基づいた時間単位のコストがあります。以下の料金と請求が別途あります：
  * GPU instanceコスト
  * 永続ストレージコスト

コンテナが実行状態にある**毎秒**、**GPU instanceコスト**が請求されます。コンテナが停止していても、コンテナが存在する**毎秒**、**永続ストレージコスト（ストレージ割り当てサイズに依存）**が請求されます。一時ストレージは現在無料です。

コンソールのBillingページで「Credit History」をクリックすると、請求履歴の詳細ドキュメントを取得できます。 [![Alt text](/img/migrated/credit-history-381407df.png)](/img/migrated/credit-history-381407df.png)
## 残高不足
アカウント残高がコンテナを1時間使用する推定総コストを下回ると、コンテナの作成、ストレージボリュームの作成、コンテナの起動、コンテナの再起動などの一部の操作が制限される場合があります。また、クレジット追加を促すシステム通知も届きます。
## マイナス残高
残高がゼロ以下になると、コンテナは自動的に停止されますが、すぐに削除されることはありません。一時ストレージのデータは削除され、復元できません。永続ストレージのデータはまだ利用可能であるため、停止中のコンテナの永続ストレージ料金は引き続き請求されます。
残高が**7日連続**でマイナスの場合、コンテナを削除し、すべてのデータが完全に削除されます。計画的にご利用ください。
