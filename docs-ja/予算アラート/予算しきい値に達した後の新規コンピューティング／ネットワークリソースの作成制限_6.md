# 予算しきい値に達した後の新規コンピューティング／ネットワークリソースの作成制限_6

予算アラートを作成するステップ4（アラートしきい値入力画面）にて[追加アクション]のオプションを選択します。
  * デフォルトオプション（ [いいえ] ）のままでは、実績コストが予算しきい値に達した後にもシステムは新規リソースの作成をブロックしません。

[![file](/img/migrated/image-1727168996714-d093ff1d.png)](/img/migrated/image-1727168996714-d093ff1d.png)
  * [はい] のオプションを選択した場合、実績コストがアラートしきい値に達した後、追加のコンピュートおよびネットワークリソース（インスタンス／VPC／ストレージ／スナップショット／IP ／LBを含む）の作成が自動的にブロックされます。

[![file](/img/migrated/image-1727169010489-9c081e43.png)](/img/migrated/image-1727169010489-9c081e43.png) [![file](/img/migrated/image-1727169025409-df35c1e8.png)](/img/migrated/image-1727169025409-df35c1e8.png)
**注記：**
  * 一つの予算内では一つのアラートにのみ追加アクションを選択できます。 追加アクションが設定されている複数の予算と複数のアラートしきい値を持つ同一テナント／VPの場合、一番最初に作動するアラートに沿って新規リソースの作成をブロックします。
