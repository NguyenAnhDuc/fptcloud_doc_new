# 予算しきい値到達時の新規リソース作成制限

Attach optionを使用すると、コストがアラートしきい値に達した際にComputeおよびNetworkリソースの追加作成を自動的にブロックできます。予算外のコスト発生を防止します。

予算アラート作成のステップ2で **Attach option** を選択します：

- **No**（デフォルト）を選択した場合：予算しきい値に達しても、新規リソースの作成はブロックされません。

  ![Attach option No](/img/migrated/image-1727168996714-d093ff1d.png)

- **Yes** を選択した場合：ステップ1で選択したリソースがアラートしきい値に達すると、追加のComputeおよびNetworkリソース（Instance、VPC、Storage、Snapshot、IP、Load Balancerを含む）の作成が自動的にブロックされます。

  ![Attach option Yes](/img/migrated/image-1727169010489-9c081e43.png)

  ![リソース作成ブロックの通知](/img/migrated/image-1727169025409-df35c1e8.png)

:::warning
- 各予算では1つのアラートにのみAttach optionを有効にできます。
- 同一テナント／VPCに複数の予算があり、複数のアラートにAttach optionが設定されている場合、最初にトリガーされたアラートがその時点でリソース作成をブロックします。
:::
