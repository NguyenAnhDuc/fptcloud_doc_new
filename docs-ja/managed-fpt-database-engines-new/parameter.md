# パラメーターの管理

### 概要

データベース管理システム（DBMS）において、パラメーターは運用中のシステムの特性と動作を調整・制御する設定値です。これらのパラメーターは、データベースシステムのパフォーマンス、メモリ、タイムアウト、最適化メカニズム、その他の側面に影響を与える可能性があります。

各パラメーターは特定の値を持ち、システム動作を最適化するためにデータベース管理者のニーズに応じて変更できます。パラメーターはDynamic（動的）またはStatic（静的）のいずれかで、それぞれ適用方法が異なります。

- **Dynamic Parameter（動的パラメーター）**：これらのパラメーターはシステムの再起動を必要とせず、データベースの実行中に変更できます。
- **Static Parameter（静的パラメーター）**：これらのパラメーターは変更を有効にするためにデータベースの再起動が必要です。

パラメーターを正しく理解・管理することは、システムが安定して効率的に動作し、ユーザーの要件を満たすことを確保するために非常に重要です。

FPT Database Engineの**パラメーター管理**機能により、ユーザーはFPT Cloud Portal上で実行中のデータベースクラスターの設定パラメーターを直接表示・更新できます。動的パラメーターを更新する場合、新しい値はサービスを中断することなく操作完了直後に適用されます。静的パラメーターを更新する場合、システムは新しい値を適用するために自動的にデータベースを再起動します。

### 必要な権限

| 権限名 | 操作種別 | 説明 |
| --- | --- | --- |
| manageDatabase:Configuration | 編集 | データベース設定（パラメーター、スケーリング設定など）を変更する権限。 |

### 更新手順

以下の手順は、データベースクラスターのパラメーター値を更新する方法を説明しています。

#### ステップ1：パラメーター管理ページへのアクセス

FPT Cloud Consoleにログインします。**Database list**ページを開き > データベースクラスターのIDを選択して詳細ページを開く > **Configure Params**タブを選択して**パラメーター管理**ページにアクセスします。

[![](/img/migrated/view-param-scaled-f9a798c8.png)](/img/migrated/view-param-scaled-f9a798c8.png)

#### ステップ2：パラメータータイプの選択

更新するパラメータータイプを選択します。

- Dynamic Params
- Static Params

#### ステップ3：パラメーター値の調整

編集するパラメーターに対応する**Edit parameter**アイコン（鉛筆アイコン）をクリックします。**Update Configure Params**ダイアログに選択されたパラメーターの完全な情報が表示され、ユーザーは許容範囲内で値を更新できます。

[![](/img/migrated/update-param-scaled-7b919278.png)](/img/migrated/update-param-scaled-7b919278.png)

パラメーター情報（データ型、最小値と最大値）を確認し、許容範囲内で「**Value**」フィールドの値を更新して「**Update**」をクリックして変更を適用します。

### ベストプラクティス

- パラメーターは少しずつ段階的に調整してパフォーマンスを監視します。
- ダウンタイムを避けるためにDynamicパラメーターを優先します。
- Staticパラメーターを変更する際はメンテナンスをスケジュールします。
- 監査と運用のために変更を記録します。
