# データベース詳細の表示

**データベースクラスター詳細表示**機能により、ユーザーは以下のことができます。

- データベースクラスターの稼働状況を監視する。
- リソース設定（CPU、RAM、Storage）を把握する。
- エンジン、バージョン、ネットワーク情報を確認する。
- システム警告（例：スケールアップ後の再起動要求）を識別する。
- 関連する管理機能にすばやくアクセスする。

この機能にアクセスするには、FPT Cloud Portalにログインします。メインメニューから**Database Platform > All Database**を選択し、データベースクラスターのIDを選択して**Overview**タブをクリックすると詳細ページが表示されます。

[![Alt text](/img/migrated/overview-scaled-11bc300a.png)](/img/migrated/overview-scaled-11bc300a.png)

データベースクラスターのすべてのコア情報が表示されます。

| セクション | 表示情報 |
| --- | --- |
| ヘッダーセクション（概要） | データベースクラスターの主要情報を表示します：クラスターID、ステータス（例：Running、Stopped）、現在の設定（vCPU – RAM – Storage）、エンジンタイプ（例：PostgreSQL、MySQL）、エンジンバージョン（例：8.0.42、17）、作成時刻。 |
| Basic Information | アプリケーション接続とインフラ確認に必要な情報を表示します：クラスター名、データベースタイプ、バージョンタイプ、エンドポイント（ドメイン/IP）、ポート、ネットワーク/サブネット。 |
| Service Information | アドオンサービスの有効/無効状態を表示します：Backup、Monitoring、Vertical Scaling。 |
| Engine Information | デフォルトデータベースへの接続情報を表示します：デフォルトデータベース名、デフォルトアカウントの認証情報（ユーザー名 – パスワード）。 |
| Data Nodes Configuration | データベースクラスター内のノード設定を表示します：ストレージポリシー、ノード数、現在のリソース設定（vCPU – RAM – Storage）。 |
| List Instances | データベースクラスター内のインスタンス（ノード）の一覧を表示します。各ノードの状態監視やトラブルシューティングに役立ちます。インスタンス名、役割（Primary = True / Replica = False）、設定、ステータス、作成時刻、最終更新時刻を含みます。 |

**データベースクラスター再起動警告：**

データベースクラスターが再起動を必要とする状態（例：コンピュートリソース調整後）の場合、ユーザーが詳細ページにアクセスすると、システムは自動的に警告ダイアログとヘッダーセクションの警告バナーを表示します。

[![Alt text](/img/migrated/overview-with-warning-popup-scaled-ffe6ebc4.png)](/img/migrated/overview-with-warning-popup-scaled-ffe6ebc4.png)

**目的：**

- クラスターは稼働中であるが、不完全な設定で動作していることを強調します。
- ユーザーが再起動を実行する前に認識・意識を持てるよう促します。

再起動が完了すると（ユーザーによるPortal上の手動再起動、またはシステムによる自動再起動いずれでも）、警告ポップアップとバナーは消えます。
