# 初期セットアップ

FPT Console PortalでFPT Database Engineサービスを使用する前に、ユーザーは以下の条件を確認する必要があります。

- サービス契約が締結されており、以下のリソースが割り当てられていること：
  - プロジェクト / テナント
  - 仮想ネットワーク（VPC）
  - ネットワーク
- FPT Console Portal上に有効なアカウントがあること。
- アカウントにDatabase Engineサービスに対する適切なアクセス権限（RBAC）が付与されていること。

**プロジェクト / テナント / VPCがない場合**は、FPT Smart Cloudの営業チームにサービス契約のお問い合わせください。連絡先情報は[FPT Smart Cloud お問い合わせ](https://fptcloud.com/lien-he/)でご確認いただけます。

**FPT Console Portalのアカウントをお持ちでない場合**は、[FPT Console Portalアカウント登録](../managed-fpt-database-engines-new/index.md)セクションの手順に従ってアカウントを登録してください。

**FPT Console Portalのアカウントをお持ちの場合**は、[FPT Console Portalへのアクセス](../managed-fpt-database-engines-new/index.md)セクションの手順に従ってアクセスしてください。

**適切なアクセス権限が付与されていない場合**は、システム管理者に権限の付与を依頼する必要があります。権限付与の手順については、[FPT Database Engineサービスアクセス権限](../managed-fpt-database-engines-new/index.md)セクションをご参照ください。

**データベースプロビジョニング時に使用するサブネットをまだ作成していない場合**は、[サブネット管理](../managed-fpt-database-engines-new/index.md)セクションの手順に従ってください。

初期セットアップ手順を完了した後、ユーザーはFPT Database Engineサービスの使用方法を理解するために[詳細ガイド](../managed-fpt-database-engines-new/index.md)に進むことができます。
