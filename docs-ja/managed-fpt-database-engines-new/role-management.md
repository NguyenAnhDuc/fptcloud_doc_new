# ロール管理

RoleはFPT Cloud Portal上のIAMモジュールのコアコンポーネントです。**ロール管理**機能により、システム管理者はFPT Database Engineサービスのユーザーに対して特定のアクセス権限を持つロールを定義・割り当てることができます。

ロールを使用することで、詳細なアクセス制御によるセキュリティの強化、最小権限の原則の適用、さまざまな運用ニーズとモデルに応じた権限の分離が可能になります。

以下の手順では、新しいRoleを作成し、対応するアクセス権限を割り当てるための詳細なガイダンスを提供します。

### ステップ1：ロール管理ページへのアクセス

FPT Cloud Portalにログインします。ログイン成功後、メインメニューから**IAM** > **Roles**を選択します。**ロール管理**インターフェースには既存のロール一覧が表示され、新規作成、編集、削除のオプションがあります。

### ステップ2：新しいロールの作成

**ロール管理**ページで**Create role**をクリックします。新規ロール作成画面が以下のように表示されます。

[![](/img/migrated/create-role-46051a0f.png)](/img/migrated/create-role-46051a0f.png)

基本情報を入力します。

- **Role name**: IAMシステムでロールを識別するために使用する名前。最大100文字で、英数字、アンダースコア（_）、ハイフン（-）、スペース、ピリオド（.）を使用できます。必須項目。
- **Description**: 目的、権限範囲、または適用するユーザーグループの説明。このフィールドにより管理と監査が明確になります。
- **Permissions**: Roleに割り当てられた権限のリスト。
  - **Permission 1**: ロールに追加済みの権限を表示します。**See more**をクリックして権限の詳細を表示し、権限設定を編集します。
  - **+ Permission**: このボタンをクリックしてロールに新しい権限を追加します。機能ごとに権限を選択できます。

権限設定の詳細については、ステップ3を参照してください。

### ステップ3：ロールの権限設定

**See more**をクリックして権限の情報フィールドを表示します。

[![](/img/migrated/see-more-permission-faeb2c18.png)](/img/migrated/see-more-permission-faeb2c18.png)

- **Service Type**: 割り当てたい権限や業務機能に対応するサービスタイプを選択します。FPT Database Engineサービスは主に _「ManageDatabase」_ と _「FDE」_ の2つのサービスタイプを使用します。
  - **ManageDatabase**: 情報の閲覧、プロビジョニング、データベースの操作、アドオンサービスの管理を含む標準的なデータベース管理活動のための権限を提供します。
  - **FDE**: 管理者アカウントパスワードの閲覧や管理など、機密性の高いデータベース関連操作のための権限を提供します。

サービスタイプを選択すると、システムは**Action**セクションに対応するすべてのアクションを自動的に表示し、選択したサービスタイプに応じて権限名を更新します。

- **Action**: ロールが実行を許可されるアクションを指定します。**See more**をクリックしてロールに割り当てるアクションを表示・選択します。選択されていないアクションは付与されず、システムによってブロックされます。
- **Resource**: ロールがアクセスを許可されるリソースを指定します。**See more**をクリックしてロールに割り当てるリソースを表示・選択します。選択されていないリソースは付与されず、システムによってブロックされます。2つのオプションがあります。
  - **All**: すべてのリソースへのアクセスを許可します。このオプションを選択すると、システムはデフォルトでロールにすべてのリソースへのアクセスを付与します。
  - **Specific**: 選択した特定のリソースへのアクセスを許可します。**注意**: このオプションで**管理者パスワード閲覧をブロックする権限**（Service TypeはFDE、アクションはFDE:hide_admin_password）を割り当てる場合、**Select resource**フィールドで制限するデータベースを選択する必要があります。選択したデータベースのみパスワード閲覧が制限され、選択されていないデータベースはパスワード閲覧が可能です。

[![](/img/migrated/resource-specified-hide-pass-ad74aea0.webp)](/img/migrated/resource-specified-hide-pass-ad74aea0.webp)

すべての情報を入力したら、**Create role**を選択してロール作成プロセスを完了します。

作成に成功すると、新しいロールが**Active**ステータスで管理リストに表示され、ユーザーへの権限付与の準備が整います。権限付与の手順については、[ユーザーグループ管理](../managed-fpt-database-engines-new/index.md)セクションをご参照ください。

必要に応じて、作成済みのロールに対して以下の操作を実行できます。

- **ロールの編集**: アクセス要件やセキュリティポリシーが変更された際に、ロールの名前、説明、権限を変更できます。**ロール管理**ページで編集するロールに対応する**Edit role**を選択します。変更を加えて**Save**をクリックして保存します。
- **ロールの削除**: 不要になったロールを削除して、アクセス権限管理システムを整理できます。**ロール管理**ページで削除するロールに対応する**Delete**を選択します。警告ダイアログで操作を確認して完了します。**警告**: **ロールを削除すると、そのロールを割り当てられているユーザーやユーザーグループのアクセス権限に影響します**。ロールが削除された後、関連する権限は即座に失効し、クラウドおよびDBaaSリソースの管理・運用が中断される可能性があります。削除を実行する前に、ロールがいかなるUser GroupまたはUserにも割り当てられていないことを確認してください。
