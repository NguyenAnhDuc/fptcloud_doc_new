# ロードバランサーの監視グラフ

ロードバランサー一覧から監視するロードバランサーをクリックします。**Monitor** タブでは、各リスナーとサーバープールのリアルタイムグラフでロードバランサーのステータスを確認できます。

**リスナーのメトリクス：**

- **[Listener] Request/s**: リスナーが 1 秒間に受け取ったリクエスト数。
- **[Listener] Response Code**: リスナーが返した HTTP ステータスコード：
  - `2xx`: 成功レスポンス（例：200 OK）
  - `3xx`: リダイレクト（例：301 Moved Permanently）
  - `4xx`: クライアントエラー（例：404 Not Found）
  - `5xx`: サーバーエラー（例：500 Internal Server Error）
- **[Listener] Data Transfer**: リスナーを通じたリアルタイムのデータ転送スループット（バイト）。

**プールのメトリクス：**

- **[Pool] Request/s**: バックエンドサーバープールへ 1 秒間に送られたリクエスト数。ロードバランサーが分散する総ワークロードを示します。
- **[Pool] Response Code**: バックエンドが返した HTTP ステータスコード。
- **[Pool] Data Transfer**: バックエンドを通じたリアルタイムのデータ転送スループット（バイト）。

**Active servers**: このロードバランサーの現在アクティブ（正常）なバックエンドサーバーの数。トラフィック需要とロードバランサーの設定に応じて変化します。
