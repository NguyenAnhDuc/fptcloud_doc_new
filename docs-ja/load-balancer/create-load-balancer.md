# ロードバランサーの作成

**手順 1:** メニューから **Load Balancer > Load Balancer** を選択し、**Create Load Balancer** をクリックします。

**手順 2:** 新しいロードバランサーの設定を入力します。

ロードバランサーを作成すると、システムは設定に基づいてデフォルトのリスナーとサーバープールを自動的に 1 つずつ作成します。

#### 1. ロードバランサー情報

| フィールド | 説明 |
| --- | --- |
| Name | ロードバランサーの名前を入力します。名前は後から変更できます。 |
| Description | 任意の説明。 |
| Size | ロードバランサーのパッケージを選択します。FPT Cloud は 4 種類の高可用性（24/7）パッケージを提供しています：Basic-1、Basic-2、Standard、Advanced。 |
| Public IP | インターネットからアクセスする場合はパブリック IP を割り当てます。 |
| Subnet | ロードバランサーのサブネットを選択するか CIDR を入力します。特定のネットワーク要件がない場合、システムが VPC 内の利用可能なサブネットを選択します。 |
| IP Private | 選択したサブネット/CIDR 内の特定のプライベート IP を入力します。特定の IP が不要な場合、システムが自動的に割り当てます。 |

:::note
ロードバランサー v2.0 では、パブリック IP を割り当てる際にロードバランサー用のセキュリティグループを作成する必要はありません。システムはリスナーの設定に基づいてファイアウォールルールを自動的に開放します。
:::

#### 2. リスナー設定

| フィールド | 説明 |
| --- | --- |
| Listener name | リスナーの名前を入力します。 |
| Description | 任意の説明。 |
| Protocol | リスナーがリクエストを受け取り転送するプロトコルを選択します。 |
| Port | リスナーがリクエストを転送するポートを選択します。 |

#### 3. サーバープール設定

| フィールド | 説明 |
| --- | --- |
| Name | サーバープールの名前を入力します。 |
| Add server | プールにメンバーを追加します。VPC 内の仮想マシンまたは外部サーバーを選択できます。 |
| Port | メンバーがリクエストを受け取るポートです。 |
| Weight | リクエスト分散のためのサーバープール内のメンバーの重みです。1 以上の値を入力します。すべてのメンバーが同等の処理能力を持つ場合、デフォルトは 1 です。 |
| Algorithm | サーバープール内のリクエスト分散アルゴリズムです。 |
| Sticky session | スティッキーセッションを有効にすると、同じクライアントからの後続リクエストが同じサーバーに転送され、セッション状態が維持されます。 |
| Health check configuration | ロードバランサーがメンバーのヘルスチェックを行う方法を設定します。デフォルト設定が自動的に適用されます。特別なヘルスチェックが不要な場合はスキップできます。 |

すべての情報を確認したら **Create** をクリックしてロードバランサーを作成します。

**手順 3:** 作成したロードバランサーのステータスを確認します。

作成後、システムはロードバランサー一覧ページにリダイレクトし、ステータスを監視できます。各ロードバランサー、リスナー、サーバープール、メンバーのステータスは以下の通りです：

| オブジェクト | プロビジョニングステータス | 動作ステータス |
| --- | --- | --- |
| ロードバランサー | **Active**: 作成完了。**Creating**: 初期化中。**Updating**: 更新中。**Deleting**: 削除中。**Pending**: 作成中にエラー発生。カスタマーサポートに連絡してください。 | **Healthy**: すべてのメンバーが正常。**Unhealthy**: 1 つ以上のメンバーが異常。**Offline**: すべてのメンバーが異常。**No healthcheck**: ヘルスチェックの設定がないか設定エラー。カスタマーサポートに連絡してください。 |
| リスナー | ロードバランサーと同様 | N/A |
| サーバープール | ロードバランサーと同様 | ロードバランサーと同様 |
| メンバー | N/A | **Healthy**: リクエストを受け取れる状態。**Unhealthy**: リクエストを受け取れない状態。 |

#### 4. 推奨アラート

ロードバランサー作成時、システムは過負荷を検知するための 2 つの重要なアラートの作成を提案します：

- ロードバランサーを通過するドロップパケット数が安全閾値（> 50 pps）を超えた場合のアラート。
- 同時アクティブ接続数がパッケージの推奨閾値を超えた場合のアラート。

アラートの受信者を選択するか、新規作成します（Email、Telegram、Slack、Teams に対応）。これらのアラートは Cloud Guard サービスと連携します。

:::note
ロードバランサーの作成には時間がかかる場合があります。ステータスが **Active** になるまでお待ちください。
:::
