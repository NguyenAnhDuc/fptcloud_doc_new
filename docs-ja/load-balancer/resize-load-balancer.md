# ロードバランサーのリサイズ

**リサイズ**機能を使用すると、使用状況に合わせてロードバランサーのパッケージを変更できます：

- **パフォーマンス向上**: アプリケーションがトラフィックを処理するのに十分なリソースを確保します。
- **可用性の向上**: トラフィックが急増しても安定したサービスを維持します。
- **コスト削減**: 実際に必要なリソースのみを使用し、無駄なコストを削減します。

**手順 1:** ロードバランサー一覧でリサイズするロードバランサーの**三点メニュー**をクリックし、**Resize** を選択します。

**手順 2:** 利用可能なロードバランサーパッケージが表示されます。要件に合ったパッケージを選択します。

**手順 3:** **Resize load balancer** をクリックして確認するか、**Cancel** で取り消します。

:::note
- リサイズ前に、VPC に新しいパッケージサイズに対応する十分なクォータがあることを確認してください。
- リサイズ処理中はロードバランサーが一時的に中断されます。処理完了後は正常に動作を再開します。
- リサイズ完了後、**Monitor** タブで実際のメトリクスを確認し、新しいパッケージが実際の使用要件を満たしていることを確認してください。
:::
