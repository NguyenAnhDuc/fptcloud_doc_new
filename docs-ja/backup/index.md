# バックアップ

## 1.1. Backup Nativeとは？
Backup Nativeは、FPT Cloud上のCloud Server（VM）とVolumeのデータバックアップサービスです。
スケジュール設定または手動でバックアップを作成し、エラー、障害、または誤操作が発生した場合に任意の時点にシステムを復元できます。
バックアップはBackup Managementインターフェースで直接管理でき、監視・操作が容易で、完全に自動化されています。

## 1.2. メリット
Backup Nativeは以下のメリットを提供します：

  * **包括的なデータ保護**
定期的なバックアップにより、VMまたはVolumeに問題が発生した場合でも安全な復旧ポイントが常に確保されます。
  * **迅速な復旧**
わずかな操作でシステムを以前の状態に戻すことができ、複雑な手動処理は不要です。
  * **完全自動化**
バックアップジョブはスケジュールに従ってバックグラウンドで実行され、稼働中のアプリケーションを中断しません。
  * **VMとVolumeの両方に対応**
アプリケーション、データベース、ファイルサーバー、dev/test環境など、多様なワークロードに対応します。
  * **管理・監視が容易**
すべてのジョブステータス、実行履歴、レポートなどが1つのインターフェースに明確に表示されます。
  * **長期運用の安心感**
操作ミス、ソフトウェア障害、または予期しない障害によるデータ損失のリスクを最小化します。

## 1.3. サポートされるリソース
Backup Nativeは現在、主に2つのリソースグループをサポートしています：

### Cloud Server（VM）
  * バックアップ時点でのVMの全状態をバックアップします。
  * サーバー上で直接稼働するアプリケーション、Webサイト、システムに適しています。

### FPT Kubernetes Engine（FKE）のPVC
  * **FKE**サービスが作成したVolumeのみバックアップジョブを作成できます。
  * K8Sクラスター上で稼働するアプリケーションデータ（データベース、ファイルストレージ、ステートフルアプリケーションなど）のバックアップをサポートします。
  * データ整合性が必要なコンテナ化ワークロードに適しています。

## 1.4. 主なユースケース
Backup Nativeがよく使われる場面：

  * **ファイルや設定を誤って削除した場合のデータ復元**
最新のバックアップコピーにVolumeを復元できます。
  * **アップデート失敗時のアプリケーションのロールバック**
新バージョンのデプロイが失敗した場合、安全なスナップショットにVMを復元できます。
  * **定期的なデータ保護**
日次/週次/月次でバックアップをスケジュールし、常に安全な復旧ポイントを確保します。
  * **大きな変更前のバックアップ**
OSのアップグレード、データベース構造の変更、ソフトウェアのアップグレードなどの前に実行します。
  * **テスト/dev環境のバックアップ保存**
dev/QAチームが機能テスト時に初期状態に簡単に戻れるようにします。
