# サービス詳細

## Client Backup Server

FPT Cloud は、VMware や Hyper-V などの仮想化システムを使用したオンプレミス環境に構成・展開されるバックアップサーバーを提供します。このバックアップサーバーは FPT Cloud の管理者によってリモートで管理・運用されます。

FPT Cloud が提供するサービス内容:

- サーバーの監視（可用性、パフォーマンス、ステータス）
- デバイスのリモート管理
- システム、OS、バックアップツールのメンテナンス

## BaaS Portal

FPT Cloud は BaaS サービスの Web ポータルへのアクセスを提供します。Portal では以下の機能を利用できます。

- 保護されているサーバーおよびシステムの監視と確認
- バックアップジョブの進捗、履歴、ステータスの監視
- バックアップサイズ、スケジュール、復元ポイントなどのサーバーシステム詳細の確認
- データの能動的な復元
- メールによるバックアップジョブの進捗通知および接続状態のリソース通知

## 用語定義

- **Agent**: 物理デバイスおよび仮想デバイスのデータ保護操作を実行するシステム内のソフトウェア。
- **VBR server**: システム内のすべてのサーバーと通信し、保護、復元、管理業務、バックアップジョブ管理、プロセスイベント管理などの活動を調整するサーバー。
- **Data Compression**: ストレージ容量を削減するデータ節約ソリューション。
- **Data Encryption**: データセキュリティソリューション。
- **Data Replication**: ユーザーシステム内のサーバーのコピーとしてデータを複製する機能。これらのコピーは即座にアクセスして高速復元に使用でき、サーバーのパフォーマンスに影響を与えることなく複数の復元ポイントの作成に利用できます。
- **Disaster Recovery**: インフラストラクチャ全体の喪失や重大な障害などの問題に対応するための計画の策定や戦略の展開。
- **Backup copy**: 同一バックアップファイルのバージョンで、長期保存のためにストレージリポジトリにコピーされたもの。ストレージリポジトリはソースシステム内に配置することも、外部に展開することも可能です。バックアップファイルはプライマリバックアップファイルと同じ形式であるため、障害発生時に必要なデータを直接復元できます。
- **Retention Period**: 特定のバックアップデータセットが保持基準を超えない期間。保持期間が終了してデータが期限切れになると、そのデータは復元に使用できなくなります。
- **Retention Points**: 連続して保存される成功したジョブの数。
- **RPO**: Restore Point Objective。バックアップスケジュールによって決定される復元ポイント。
- **RTO**: Restore Time Objective。復元方法と復元するデータ量によって決定される復元時間。
