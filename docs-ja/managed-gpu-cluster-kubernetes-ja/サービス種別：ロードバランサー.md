# サービス種別：ロードバランサー

マネージドGPUクラスター製品は、Kubernetes Nativeから開発され、FPT Cloud ControllerManagerなど、より多くのクラウドプロバイダーコンポーネントをKubernetesに統合します。このコンポーネントは、クラスターおよびロードバランサーサービス内のワーカーノードを管理することを目的としています。アプリケーションやサービスにユーザーがアクセスできるように、アプリケーションをインターネットに公開できます。これには、サービスのイングレスの作成、ノードポートとしてのサービスの作成、ワーカーノードへのフローティングのアタッチ、またはロードバランサーサービスの使用など様々な方法があります。
FPTCloudは、オプション付きロードバランサーサービスの作成をサポートします。

さらに、マネージド GPU クラスターでは、ユーザーが次の構成を行うことができます。
#### 1. ロードバランサーにアタッチするフローティングIPを指定するロードバランサーサービスタイプの作成

```
Copykind: Service
apiVersion: v1
metadata:
  name: hello-world
  namespace: example
spec:
  type: LoadBalancer
  loadBalancerIP: 103.160.90.207
  selector:
    app: hello-world
  ports:
    - name: http
      protocol: TCP
      port: 443
      targetPort: web
```

**注:** パブリック IP は VPC に割り当て、非アクティブ状態にする必要があります。ユーザーは**ネットワーキング - >フローティングIPs** で確認できます。
#### 2. 設定してロードバランサーへのアクセスを制限するサービス構成の "spec" の "loadBalancerSourceRanges" :

```
Copykind: Service
apiVersion: v1
metadata:
  name: hello-world
  namespace: example1
spec:
  loadBalancerSourceRanges:
  - 14.233.234.0/24
  - 10.250.0.0/24
  type: LoadBalancer
  selector:
    app: hello-world
  ports:
    - name: http
      protocol: TCP
      port: 80
      targetPort: web
```

注: 「loadBalancerSourceRanges」構成には、ロードバランサーへのアクセスが許可されているパブリック IP 範囲の配列が含まれています。デフォルトでは、M-FKE は IPソース範囲の 0.0.0.0/0 のロードバランササービスタイプを作成します。
